# 【PRD】AI 软硬件全栈能力验证平台

## 0. 文档说明

本文档用于指导 AI 软硬件全栈能力验证平台的一期产品设计、前后端开发和后续 HTML 原型制作。

本版范围不包含运营后台、付费、额度购买、社区页面设计。社区仅作为跳转入口保留。

本版不设计单独的“芯片验证任务”。芯片相关能力体现在算力资源、任务目标硬件、验证报告指标和首页验证排行榜中。

## 1. 产品概述

### 1.1 平台定位

AI 软硬件全栈能力验证平台面向模型、算子、芯片和算力环境的验证需求，为用户提供任务创建、任务执行、结果采集、报告生成、智能分析、资源管理和团队协作能力。

平台支持用户通过表单或自然语言创建验证任务，选择平台内置验证能力或多 Agent 验证系统执行任务，并在任务完成后自动生成网页形式的验证报告。

### 1.2 目标用户

| 用户类型 | 典型诉求 |
| --- | --- |
| 基础架构研发团队 | 验证模型、算子、软件栈和芯片/算力资源的适配效果 |
| 模型研发团队 | 对比模型在不同芯片和算力环境下的训练、推理表现 |
| AI 应用团队 | 选择适合具体业务场景的模型、芯片和算力资源 |
| 技术采购团队 | 基于验证结果进行芯片、算力、模型选型决策 |

### 1.3 核心能力

| 能力 | 说明 |
| --- | --- |
| 全栈验证 | 支持模型、算子、芯片指标和算力环境组合验证 |
| 动态场景评测 | 按大语言、多模态、音频、科学计算、搜广推等场景组织验证能力 |
| Agent 集成 | 支持将任务下发至多 Agent 验证系统，接收状态、日志、指标和结果 |
| 报告与建议 | 自动生成基础报告，并支持智能分析报告输出原因分析、改进建议和选型建议 |
| 资源协作 | 支持个人、团队、公共资产，以及算力资源接入和团队授权引用 |

### 1.4 一期目标

一期重点实现：

1. 评测任务创建与管理。
2. 验证执行与调度。
3. 算力资源管理和用户自有算力接入。
4. 评测报告管理。
5. 数字资产管理。
6. 用户、团队和资源共享管理。
7. SkillHub 入口。

一期验证目标：

1. 支持 10 个模型。
2. 支持 5 款国产芯片 + 英伟达芯片。
3. 支持 5 个场景的能力验证。
4. 支持生成基础报告和智能分析报告。

## 2. 登录、注册与用户管理

### 2.1 注册与登录页面

#### 2.1.1 页面结构

页面采用左右分栏布局。

左侧为产品能力展示区，包含：

| 元素 | 说明 |
| --- | --- |
| 平台 Logo | 展示验证平台品牌标识 |
| 平台名称 | AI 软硬件全栈能力验证平台 |
| 能力摘要 | 模型验证、算子验证、芯片指标、算力接入、报告生成、智能分析 |

右侧为登录/注册卡片，默认展示登录状态。

#### 2.1.2 登录方式

登录卡片支持三种方式：

| 登录方式 | 字段 | 说明 |
| --- | --- | --- |
| 实验室 SSO | SSO 登录按钮 | 点击后跳转或唤起实验室 SSO 登录流程 |
| 验证码登录 | 手机号、验证码、记住登录状态 | 未注册手机号通过验证码登录后自动注册 |
| 账号密码登录 | 手机号/邮箱/用户名、密码、记住登录状态 | 已注册用户使用密码登录 |

交互要求：

1. 默认展示验证码登录。
2. 用户可通过 Tab 切换验证码登录、账号密码登录。
3. 登录按钮启用条件：账号字段不为空；验证码登录时验证码不为空；密码登录时密码不为空。
4. 记住登录状态勾选后，登录有效期延长 14 天。
5. 未注册手机号通过验证码登录时，用户必须勾选同意用户协议。
6. 登录成功后，系统校验账号状态，获取用户可访问工作空间列表，并默认进入个人工作空间。

#### 2.1.3 注册方式

注册支持手机号注册和邮箱注册。

手机号注册字段：

| 字段 | 是否必填 | 说明 |
| --- | --- | --- |
| 用户名 | 是 | 支持英文、数字和连字符，设置后不可更改 |
| 手机号 | 是 | 用于注册和接收验证码 |
| 验证码 | 是 | 手机验证码 |
| 设置密码 | 是 | 用户登录密码 |
| 确认密码 | 是 | 与设置密码一致 |
| 用户协议 | 是 | 勾选后才允许注册 |

邮箱注册字段：

| 字段 | 是否必填 | 说明 |
| --- | --- | --- |
| 用户名 | 是 | 支持英文、数字和连字符，设置后不可更改 |
| 邮箱 | 是 | 用于注册和接收验证码 |
| 验证码 | 是 | 邮箱验证码 |
| 设置密码 | 是 | 用户登录密码 |
| 确认密码 | 是 | 与设置密码一致 |
| 用户协议 | 是 | 勾选后才允许注册 |

交互要求：

1. 用户在密码登录卡片点击“注册”进入注册状态。
2. 注册成功后自动返回登录卡片。
3. 用户可点击“返回登录”取消注册。
4. 前端需校验手机号、邮箱、密码一致性和必填状态。
5. 后端需校验用户名、手机号、邮箱唯一性。

### 2.2 工作空间账号体系

平台采用“个人账号 + 工作空间”的账号体系。

1. 用户注册成功后，系统自动创建个人工作空间。
2. 用户可创建或加入团队工作空间。
3. 个人工作空间承载个人任务、个人资产、个人报告。
4. 团队工作空间承载团队任务、团队资产、团队报告。
5. 系统底层不采用主账号/子账号强绑定关系，而是采用独立账号 + 团队角色的方式管理权限。

### 2.3 个人账号管理

入口位于右上角头像菜单，所有用户均可访问。

页面包含以下模块：

| 模块 | 功能 |
| --- | --- |
| 个人资料 | 查看和修改头像、昵称、姓名、公司/组织、职位 |
| 偏好设置 | 设置语言、主题、默认工作空间 |
| 安全设置 | 绑定/更换手机号、邮箱，修改密码 |
| 登录管理 | 查看当前登录状态，退出登录 |

交互要求：

1. 修改头像支持上传图片，前端需限制图片格式和大小。
2. 修改手机号或邮箱需验证码确认。
3. 修改密码需输入旧密码、新密码、确认新密码。
4. 退出登录后返回登录页面。

### 2.4 团队管理

团队管理仅在团队工作空间内展示，且仅团队拥有者或管理员可访问。

团队管理包含以下 Tab：

| Tab | 功能 |
| --- | --- |
| 团队信息 | 查看和编辑团队名称、团队描述 |
| 成员管理 | 查看成员列表、邀请成员、移除成员、修改角色 |
| 邀请管理 | 生成邀请码、复制邀请码、重置邀请码 |
| 资源共享管理 | 查看团队可用资产来源，管理授权引用关系 |
| 任务与报告权限 | 查看团队任务和报告的权限规则 |

团队角色：

| 角色 | 权限 |
| --- | --- |
| 团队拥有者 | 管理团队信息、成员、角色、资源、任务、报告，可移交或解散团队 |
| 团队管理员 | 管理成员、资源、任务、报告，不可解散团队 |
| 团队成员 | 使用团队资源，创建任务，查看授权范围内的任务和报告 |

成员管理交互：

1. 邀请成员时可输入手机号、邮箱或用户名。
2. 邀请时需选择角色，默认团队成员。
3. 移除成员需二次确认。
4. 修改角色后即时生效。
5. 成员被移除后，不再拥有团队资产、任务、报告访问权限。

## 4. 评测任务创建

### 4.1 入口与分类

用户可通过以下入口创建任务：

1. 首页快捷入口“新建任务”。
2. 左侧导航“评测任务 - 任务创建”。
3. 任务管理列表页“新建任务”按钮。
4. 自然语言创建任务入口。

任务创建分类：

1. 模型验证。
2. 算子验证。
3. 自然语言创建任务。

本版不提供单独芯片验证任务入口。

### 4.2 模型验证任务

模型验证支持训练验证和推理验证。

页面结构：

| 区域 | 内容 |
| --- | --- |
| 基础信息 | 任务名称、标签 |
| 验证目标 | 训练验证 / 推理验证 |
| 模型选择 | 公共模型 / 我的模型 / 团队模型 |
| 数据集选择 | 公共数据集 / 我的数据集 / 团队数据集 |
| 算力资源 | 目标芯片、算力资源 |
| 镜像配置 | 公共镜像 / 我的镜像 / 团队镜像 |
| 执行方式 | 平台内置验证能力 / 多 Agent 验证系统 |
| 操作区 | 确定、取消 |

表单字段：

| 字段 | 输入方式 | 是否必填 | 说明 |
| --- | --- | --- | --- |
| 任务名称 | 文本框 | 是 | 验证任务名称 |
| 标签 | 文本框或标签选择 | 否 | 用于任务分类 |
| 验证目标 | 单选 | 是 | 训练验证或推理验证 |
| 模型 | 搜索下拉单选 | 是 | 可选择公共、我的、团队模型 |
| 数据集 | 搜索下拉单选 | 是 | 可选择公共、我的、团队数据集 |
| 算力资源 | 搜索下拉单选 | 是 | 仅展示可用且可调度资源 |
| 镜像 | 搜索下拉单选 | 是 | 与模型和算力环境匹配 |
| 执行方式 | 单选 | 是 | 平台内置验证能力或多 Agent 验证系统 |

指标规则：

1. 用户不需要在创建任务时手动选择验证指标。
2. 模型验证任务默认关联模型验证指标 + 芯片指标。
3. 模型推理指标包含吞吐量、延迟、准确率、资源消耗等。
4. 模型训练指标包含吞吐量、能效比、MFU、收敛表现、资源消耗等。
5. 芯片指标包含计算精度、实测算力、显存带宽、能效比等。

校验规则：

1. 必填字段不能为空。
2. 模型、数据集、算力资源、镜像必须各选择一项。
3. 算力资源必须处于可用且可调度状态。
4. 镜像需与任务类型和算力环境兼容。

提交成功后：

1. 后端创建任务并返回任务 ID。
2. 任务进入排队中状态。
3. 页面跳转任务详情页或任务管理列表。
4. 系统 Toast 提示“任务已创建”。

页面结构：

| 区域 | 内容 |
| --- | --- |
| 基础信息 | 任务名称、标签 |
| 算子选择 | 预置算子 / 我的算子 / 团队算子 |
| 算力资源 | 目标芯片、算力资源 |
| 镜像配置 | 公共镜像 / 我的镜像 / 团队镜像 |
| 执行方式 | 平台内置验证能力 / 多 Agent 验证系统 |
| 操作区 | 确定、取消 |

表单字段：

| 字段 | 输入方式 | 是否必填 | 说明 |
| --- | --- | --- | --- |
| 任务名称 | 文本框 | 是 | 验证任务名称 |
| 标签 | 文本框或标签选择 | 否 | 用于任务分类 |
| 算子 | 级联选择或搜索下拉 | 是 | 可选择单个或多个算子 |
| 算力资源 | 搜索下拉单选 | 是 | 仅展示可用且可调度资源 |
| 镜像 | 搜索下拉单选 | 是 | 与算子和算力环境匹配 |
| 执行方式 | 单选 | 是 | 平台内置验证能力或多 Agent 验证系统 |

指标规则：

1. 用户不需要在创建任务时手动选择验证指标。
2. 算子验证任务默认关联算子验证指标 + 芯片指标。
3. 算子指标包含功能完备性、单算子执行耗时、吞吐量、执行稳定性等。
4. 芯片指标包含计算精度、实测算力、显存带宽、能效比等。

### 4.4 任务执行方式选择

执行方式为任务创建必选项。

| 执行方式 | 说明 | 适用场景 |
| --- | --- | --- |
| 平台内置验证能力 | 使用平台预置验证流程执行任务 | 标准模型/算子验证 |
| 多 Agent 验证系统 | 将任务下发至多 Agent 验证系统执行 | 需要复杂环境、场景推理、智能分析输入的任务 |

交互要求：

1. 默认选中平台内置验证能力。
2. 多 Agent 验证系统不可用时置灰，并展示不可用原因。
3. 选择多 Agent 验证系统后，任务详情需展示 Agent 执行状态和返回结果。

### 4.5 自然语言创建任务

入口：用户点击“评测任务 - 自然语言创建任务”。

页面形式：右侧弹出交互式对话框，非全屏弹窗。

流程：

1. 用户在输入框输入自然语言任务描述。
2. 点击“提交”。
3. 系统解析任务类型、模型/算子、数据集、算力资源、镜像、执行方式等字段。
4. 系统在对话框内生成可编辑表单。
5. 已提取字段高亮显示，未提取字段置灰。
6. 若缺少必填字段，系统在表单下方内联提示用户补充。
7. 用户补充信息后，系统增量解析，不重置已填字段。
8. 用户确认表单后点击“创建任务”。
9. 系统创建任务并启动执行流程。

冲突检测示例：

1. 用户输入“用 MMLU 测速度”。
2. 系统识别 MMLU 不支持速度指标。
3. 提示“MMLU 主要用于准确率验证，是否改为准确率验证？”

创建成功后：

1. 关闭对话框。
2. 任务管理列表实时刷新并高亮新任务。
3. Toast 提示“任务[名称]已启动”。
4. 返回任务 ID 和预计完成时间。

### 5.1 任务列表

入口：左侧导航“评测任务 - 任务管理”。

页面结构：

| 区域 | 内容 |

| 搜索区 | 按任务名称或任务 ID 搜索 |

| 分页区 | 页码、每页条数 |

筛选规则：

1. 支持按状态筛选：排队中、运行中、已完成、失败、终止。
2. 支持按任务类型筛选：模型验证、算子验证。
3. 支持按执行方式筛选：平台内置验证能力、多 Agent 验证系统。
4. 支持按创建时间范围筛选。
5. 任意筛选条件变更后即时刷新列表。
6. 提供重置按钮恢复默认筛选条件。

任务列表精简字段：

| 字段 | 说明 |
| --- | --- |

| 任务类型 | 模型验证 / 算子验证 |
| 执行方式 | 平台内置 / 多 Agent |

| 创建时间 | 任务创建时间 |
| 操作 | 详情、复制、删除、终止、重试、下载日志 |

### 5.2 批量操作

批量删除：

1. 初始置灰。
2. 至少选择一个任务后启用。
3. 点击后弹出二次确认：“确认删除选中的 N 个任务？操作不可恢复。”
4. 仅允许删除已完成、失败、终止、排队中的任务。
5. 若包含运行中任务，提示“含有无法删除的运行中任务，请先终止”。

批量终止：

1. 初始置灰。
2. 选中至少一个运行中任务后启用。
3. 点击后弹出二次确认：“将对选中的 N 个任务中符合条件的 M 个执行终止，是否继续？”
4. 确认后仅终止运行中任务，其他状态不变。
5. 操作完成后提示“已成功终止 M 个任务”。

### 5.3 单任务操作

| 操作 | 显示规则 | 说明 |
| --- | --- | --- |
| 详情 | 始终显示 | 进入任务详情页 |
| 复制 | 始终显示 | 跳转任务创建页并填充原任务配置，任务名称追加“_副本” |
| 删除 | 非运行中任务显示 | 二次确认后删除任务 |
| 终止 | 排队中、运行中显示 | 终止任务 |
| 重试 | 失败、终止显示 | 使用原配置重新提交任务 |
| 下载日志 | 有日志时显示 | 下载任务执行日志 |

### 5.4 任务详情

任务详情页面结构：

| 区域 | 内容 |
| --- | --- |
| 顶部信息栏 | 任务名称、任务 ID、状态、任务类型、执行方式、创建人、创建时间 |
| 配置摘要 | 模型/算子、数据集、算力资源、镜像、标签 |
| 执行详情 | 状态流转、进度、关键节点时间 |
| 日志区 | 执行日志、Agent 日志、错误信息 |

| 操作区 | 复制、删除、终止、重试、下载日志、返回列表 |

任务状态流转：

1. 排队中：任务已创建，等待资源分配。
2. 运行中：任务正在执行。
3. 已完成：任务执行成功。
4. 失败：任务执行失败。
5. 终止：任务被用户或系统终止。

## 6. 验证执行与调度

### 6.1 总体流程

1. 用户创建验证任务。
2. 后端保存任务配置。
3. 调度器根据执行方式选择执行通道。
4. 系统分配算力资源。
5. 执行通道运行验证任务。
6. 执行过程中回传状态、日志、进度、指标。
7. 任务完成后回传结果。
8. 系统自动开始生成基础报告。

### 6.2 任务调度器

任务调度器负责：

| 能力 | 说明 |
| --- | --- |
| 路由任务 | 根据执行方式路由至平台内置验证能力或多 Agent 验证系统 |
| 队列管理 | 管理排队中任务 |
| 资源分配 | 选择可用且可调度算力资源 |
| 状态同步 | 更新任务状态、进度、关键时间 |
| 终止任务 | 接收用户终止操作并通知执行通道 |
| 重试任务 | 基于原配置重新提交任务 |

### 6.3 平台内置验证能力

平台内置验证能力用于执行标准验证流程。

后端要求：

1. 接收任务配置。
2. 准备模型、数据集、算子、镜像和算力环境。
3. 执行预置验证脚本或流程。
4. 回传任务状态、日志、指标和结果。
5. 支持任务终止。
6. 支持失败后重试。

### 6.4 多 Agent 验证系统

多 Agent 验证系统用于复杂验证任务和智能分析输入。

后端要求：

1. 将任务配置转换为多 Agent 系统可识别的任务描述。
2. 下发任务。
3. 接收 Agent 执行状态。
4. 接收 Agent 日志、指标和结果。
5. 接收用于智能分析报告的分析上下文。
6. 支持任务终止和状态同步。

任务详情展示要求：

1. 展示执行方式为“多 Agent 验证系统”。
2. 展示 Agent 执行状态。
3. 展示 Agent 返回日志。
4. 展示 Agent 返回结果摘要。

### 6.5 轻量级评测任务执行代理

轻量级执行代理部署在外部资源侧。

代理职责：

1. 接收平台下发的任务。
2. 准备执行环境。
3. 拉取或挂载模型、数据集、算子、镜像。
4. 执行验证任务。
5. 采集日志、指标、结果。
6. 回传任务状态和结果。
7. 接收平台终止指令。
8. 定期上报心跳。

代理状态：

| 状态 | 说明 |
| --- | --- |
| 未绑定 | 资源尚未绑定代理 |
| 在线 | 代理心跳正常 |
| 离线 | 超过心跳阈值未上报 |
| 异常 | 代理可连接但检测异常 |

| 停用 | 用户或管理员主动停用 |

任务创建限制：

1. 任务创建时仅可选择可用且可调度的算力资源。
2. 代理离线的资源不可被选择。
3. 检测失败、待审核、停用资源不可被选择。
4. 不可选资源可显示但置灰，并展示不可用原因。

### 9.2 公共资产

公共资产由平台统一维护，用户只可查看和引用，不可编辑或删除。

公共资产包括：

1. 平台预置模型。
2. 平台预置数据集。

公共资产列表支持：

2. 标签筛选。
3. 模糊搜索。
4. 查看详情。
5. 在任务创建中引用。

我的资产为当前用户上传、创建或维护的私有资产。

支持操作：

| --- | --- |
| 新建/上传 | 上传模型、数据集、算子、镜像 |
| 查看详情 | 查看资产基本信息和使用记录 |
| 编辑 | 修改名称、描述、标签等元信息 |
| 删除 | 删除本人拥有的资产 |
| 共享 | 授权团队引用该资产 |

### 9.4 团队资产

团队资产包含：

1. 团队拥有的资产。
2. 个人授权引用给团队的资产。

团队资产列表需展示资产来源：

| 来源 | 说明 |
| --- | --- |
| 团队自有 | 资产归属团队 |
| 成员授权 | 资产归属个人，团队获得引用权限 |
| 公共引用 | 公共资产在团队任务中被引用 |

### 9.5 资产共享：授权引用

个人资产共享给团队采用授权引用机制。

规则：

1. 不复制底层数据或文件。
2. 资产归属仍为个人 owner。
3. 团队成员获得可见和可引用权限。
4. owner 可取消共享。
5. 取消共享后，团队成员不可再在新任务中引用该资产。
6. 已经使用该资产创建的历史任务和报告保留历史引用记录。

共享流程：

1. 用户在“我的资产”点击“共享”。
2. 选择目标团队。
3. 选择共享权限：可查看、可引用。
4. 确认共享。
5. 团队资产 Tab 中出现该资产，来源标记为“成员授权”。

取消共享流程：

1. owner 在资产详情或共享管理中点击“取消共享”。
2. 系统提示影响范围。
3. 用户确认后取消授权。
4. 团队侧立即不可见或不可引用。

### 9.6 资产审核上架

管理员可审核用户上传或申请上架的资源。

审核状态：

| 状态 | 说明 |
| --- | --- |
| 草稿 | 用户已创建但未提交 |
| 待审核 | 已提交审核 |
| 审核通过 | 可进入指定可见范围 |
| 审核驳回 | 不可上架，需展示驳回原因 |

审核通过后，可见范围包括：

1. 仅个人。
2. 指定团队。
3. 公共资产池。

权限显隐要求：

1. 团队管理仅团队拥有者和管理员可见。
2. 资源共享管理仅 owner、团队管理员和团队拥有者可操作。
3. 自有算力接入入口对有权限的个人用户和团队管理员开放。
4. 删除、停用、取消共享等高风险操作需二次确认。

## 13. 验收标准

1. 用户可完成登录/注册，并进入个人工作空间。
2. 有团队时可切换团队空间，无团队时隐藏空间切换器。
3. 首页可按五个场景查看芯片验证排行榜，并可点击跳转社区排行榜。
4. 首页最新产品动态可点击“了解更多”跳转社区。
5. 任务创建不出现优先级字段，也不要求用户手动选择验证指标。
6. 模型验证任务自动关联模型指标与芯片指标。
7. 算子验证任务自动关联算子指标与芯片指标。
8. 不存在单独芯片验证任务入口。
9. 用户可选择平台内置验证能力或多 Agent 验证系统执行任务。
10. 个人资产授权给团队后，团队成员可在团队空间中查看并在任务中引用。
11. 取消授权后，团队成员不可再在新任务中引用该资产。
12. 用户可在算力资源模块登记自有算力，完成代理绑定和检测后进入可调度资源池。
13. 算力资源列表保持精简，详情页可查看完整连接、代理、审核和能力信息。
14. 任务完成后自动生成基础报告。
15. 用户可在基础报告详情页生成自定义报告。
16. 报告详情以网页形式展示，支持导出下载到本地。
17. 智能分析报告包含原因分析、改进建议和选型建议。
18. HTML 原型覆盖从登录/注册到任务、报告、算力、资产、用户管理的完整页面链路。
